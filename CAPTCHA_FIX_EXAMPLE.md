```json
{

  "challenge": {
    "token": "abc123...",
    "difficulty": 1,
    "expires_at": "2026-05-17T05:32:00Z",
    "payload": {
      "prompt": "Сколько будет 5 + 3?",
      "map": {"A": "5", "B": "3"},
      "steps": ["Сложите A и B", "Введите результат"]
    }
  }
}
```

**Стало (API возвращает):**
```json
{
  "ok": true,
  "challenge": {
    "token": "abc123...",

```

## Проверка работы

### 1. Откройте консоль браузера (F12)

При загрузке капчи вы должны увидеть:
```
Captcha question: Сколько будет 5 + 3?
```

### 2. Проверьте элементы страницы

Должны быть видны:
- ✅ Блок с вопросом (синяя рамка)
- ✅ Блок с подсказками (или "Подсказок нет")
- ✅ Блок с условиями (или "Условий нет")
- ✅ Поле ввода
- ✅ Кнопка "Проверить"
- ✅ Таймер обратного отсчета

### 3. Введите ответ

- Введите правильный ответ
- Нажмите "Проверить" или Enter
- Должно появиться: "✅ Капча пройдена! Ограничение снято."
- Через 2 секунды webapp закроется

## Что делать если капча все еще не работает

### 1. Проверьте что webapp пересобран
```bash
cd webapp/ui_from_testpers
npm run build
```

### 2. Очистите кеш браузера
- Откройте webapp
- Нажмите Ctrl+Shift+R (жесткая перезагрузка)

### 3. Проверьте логи бота
```bash
tail -f bot.log | grep captcha
```

### 4. Проверьте консоль браузера (F12)
Должны быть логи:
```
Captcha question: ...
```

Если есть ошибки - скопируйте их для отладки

### 5. Проверьте API
Откройте в браузере (замените TOKEN):
```
https://your-domain.com/api/captcha/challenge?token=TOKEN
```

Должен вернуться JSON с полями:
- `ok: true`
- `challenge.question`
- `challenge.map`
- `challenge.steps`
