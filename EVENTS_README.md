# Система событий на локациях

## Как работают события

### Глобальная механика (как эко-катастрофа)

События работают **для всей локации**, а не для отдельных игроков:

1. **Проверка при каждой рыбалке**
   - Когда ЛЮБОЙ игрок ловит рыбу на локации
   - Система проверяет: есть ли активное событие?
   - Если нет - бросается кубик с шансом запуска

2. **Запуск события**
   - Событие появляется на локации (например, "Нерест" на "Море")
   - ВСЕ игроки на этой локации попадают под его эффект
   - Событие длится 30-120 минут (зависит от типа)

3. **Перемещение события**
   - После окончания события есть cooldown (6-12 часов)
   - После cooldown событие может появиться на ДРУГОЙ локации
   - Например: "Убийство" было на "Море" → закончилось → через 12ч появилось на "Реке"

4. **Глобальное влияние**
   - Все игроки видят одно и то же событие на локации
   - Команда `/weather` показывает текущее событие
   - События создают "горячие точки" для рыбалки

### Частота появления

**НЕ "1 раз в 1000 забросов для игрока"**, а:

- При каждой рыбалке на локации проверяется шанс 0.03%-0.1%
- Если на локации рыбачат 100 игроков в час → событие появится довольно быстро
- Если локация пустая → событие может не появляться долго
- **Активность игроков влияет на частоту событий!**

### Пример жизненного цикла события

```
10:00 - Игрок A ловит рыбу на "Море" → проверка → событие НЕ появилось
10:05 - Игрок B ловит рыбу на "Море" → проверка → событие НЕ появилось
10:10 - Игрок C ловит рыбу на "Море" → проверка → ✅ СОБЫТИЕ "Нерест" появилось!

10:10-11:40 - ВСЕ игроки на "Море" получают бонус +20% к улову
              Игроки A, B, C, D, E... все видят "Нерест" в /weather

11:40 - Событие "Нерест" закончилось
11:40-17:40 - Cooldown 6 часов (нельзя запустить "Нерест" нигде)

17:40 - Cooldown закончился
18:00 - Игрок X ловит рыбу на "Реке" → проверка → ✅ "Нерест" появился на "Реке"!
18:00-19:30 - ВСЕ игроки на "Реке" получают бонус

19:30 - Событие закончилось, и цикл повторяется...
```

## Изменения

### 1. Изменение cooldown команды /pray
- **Было:** раз в 8 часов
- **Стало:** раз в 24 часа

### 2. Новые события на локациях

#### 🐟 Нерест
- **Описание:** Рыба активно клюёт! Шанс поймать рыбу (не мусор) увеличен на +20%
- **Шанс появления:** 0.1% при каждой рыбалке
- **Длительность:** 45-90 минут
- **Cooldown:** 6 часов после окончания
- **Эффект:** Бонус +20% конвертируется в +200 к roll, что увеличивает шанс поймать рыбу вместо мусора

#### ☠️ Убийство
- **Описание:** В воде замечена только одна особь... Ловится только один конкретный вид рыбы
- **Шанс появления:** 0.03% (очень редкое событие!)
- **Длительность:** 30-60 минут
- **Cooldown:** 12 часов после окончания
- **Эффект:** 
  - Система случайно выбирает одну рыбу из доступных на локации
  - Во время события ловится ТОЛЬКО эта рыба
  - Игнорируются редкости и наживки - всегда эта конкретная рыба
  - Отлично для целенаправленной ловли редких видов

#### 🐠 Стайный инстинкт
- **Описание:** Огромная стая одного вида заходит в воду. Ловите их подряд для бонуса к весу!
- **Шанс появления:** 0.08%
- **Длительность:** 60-120 минут
- **Cooldown:** 8 часов после окончания
- **Механика:**
  - Система выбирает один вид рыбы для "стаи"
  - При ловле этой рыбы начинается цепочка
  - Каждая последующая рыба того же вида дает **+10% к весу**
  - Максимальный бонус: **+50%** (после 5 рыб подряд)
  - **ВАЖНО:** Если поймали другую рыбу - цепочка сбрасывается!
  - Цепочка также сбрасывается при смене локации
- **Стратегия:**
  - Используйте прикормку и наживку для нужного вида
  - Терпеливо отпускайте "не ту" рыбу
  - Цель: набить максимальную цепочку для рекордов по весу

## Команды

### /events
Просмотр активных событий на текущей локации

**Для владельцев бота:**
- `/events start <тип> <локация>` - запустить событие
  - Типы: `spawn`, `murder`, `school`, `eco`
- `/events stop <тип> <локация>` - остановить событие

### /weather
Обновлена команда - теперь показывает все активные события на локации

### /pray
Изменен cooldown с 8 часов на 24 часа

### Новые таблицы в БД

#### location_events
```sql
CREATE TABLE location_events (
    id SERIAL PRIMARY KEY,
    event_type TEXT NOT NULL,
    location TEXT NOT NULL,
    started_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    ends_at TIMESTAMP NOT NULL,
    is_active INTEGER DEFAULT 1,
    params TEXT DEFAULT '{}'  -- JSON с параметрами события
)
```

#### school_chains
```sql
CREATE TABLE school_chains (
    id SERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL,
    event_id BIGINT NOT NULL,
    location TEXT NOT NULL,
    fish_name TEXT NOT NULL,
    chain_count INTEGER NOT NULL DEFAULT 0,
    last_catch_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, event_id)
)
```

## Совместимость

- Совместимо со старой системой эко-катастроф
- Все старые функции работают как прежде
- **ОГРАНИЧЕНИЕ:** На одной локации может быть только ОДНО событие
- **ОГРАНИЧЕНИЕ:** Каждый тип события может быть только на ОДНОЙ локации
- Только эко-катастрофа остается эксклюзивной (одна на все локации, старая система)

## Стратегическое значение

### Для сообщества игроков

1. **"Горячие точки"**
   - Игроки узнают где активно событие
   - Все стремятся на эту локацию
   - Создается естественная концентрация игроков

2. **Обмен информацией**
   - "На Море сейчас Нерест!"
   - "Убийство на Реке - ловится только Щука!"
   - Социальный аспект игры

3. **Динамика локаций**
   - События мигрируют между локациями
   - Игроки путешествуют за событиями
   - Нет застоя на одном месте

### Благодаря ограничениям

- **Редкость** - событие на одной локации в момент времени
- **Ценность** - игроки ценят время когда есть событие
- **Стратегия** - выбор локации важен
- **Баланс** - нельзя "фармить" несколько событий сразу

### Особенность "Стайного инстинкта"

Хотя событие глобальное для локации, **цепочки уникальны для каждого игрока**:
- Событие "Стая Плотвы" активно на "Море" (все видят)
- Игрок A поймал 3 плотвы подряд → цепочка 3, бонус +30%
- Игрок B поймал 1 плотву, потом окуня → цепочка сброшена
- Каждый игрок управляет своей цепочкой независимо

## Баланс

### Шансы и частота

События настроены так, чтобы быть редкими но реальными:

| Событие | Шанс при рыбалке | Частота на активной локации* |
|---------|------------------|------------------------------|
| Нерест | 0.1% (1 из 1000) | ~1 раз в 1-2 часа |
| Убийство | 0.03% (1 из 3333) | ~1 раз в 3-5 часов |
| Стайный инстинкт | 0.08% (1 из 1250) | ~1 раз в 1-3 часа |

\* При условии что на локации рыбачат ~100 игроков в час

### Длительность и cooldown

| Событие | Длительность | Cooldown | Итого цикл |
|---------|--------------|----------|------------|
| Нерест | 45-90 мин | 6 часов | ~7-8 часов |
| Убийство | 30-60 мин | 12 часов | ~13 часов |
| Стайный инстинкт | 60-120 мин | 8 часов | ~9-10 часов |

### Влияние активности

**На популярной локации** (100+ рыбалок в час):
- События появляются регулярно
- Игроки часто видят бонусы
- Высокая конкуренция за событие

**На тихой локации** (10-20 рыбалок в час):
- События редкие
- Можно "фармить" в одиночку
- Меньше конкуренции

**На заброшенной локации** (<5 рыбалок в час):
- События почти не появляются
- Можно ждать очень долго
- Стимул к социальной игре

Cooldown обеспечивает, что:
- События не надоедают
- Есть перерывы между появлениями
- Мигрируют между локациями
