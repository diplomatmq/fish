# Чеклист тестирования команды /send

## Базовые проверки

- [ ] `/send` работает только в личных сообщениях (в группах бот игнорирует)
- [ ] `/send` показывает меню с 5 типами подарков
- [ ] `/cancel` отменяет процесс отправки с сообщением "✅ Ваша попытка отправить подарок отменена."
- [ ] `/cancel` не показывает лишние сообщения про RAF

## Отправка монет

- [ ] Выбор "💰 Монеты" → запрос количества
- [ ] Ввод числа → запрос username/ID
- [ ] Ввод username → отправка инвойса 25 звезд
- [ ] Оплата → монеты списываются у отправителя, добавляются получателю
- [ ] Проверка: недостаточно монет → ошибка
- [ ] Проверка: отрицательное число → ошибка
- [ ] Проверка: невалидный текст вместо числа → ошибка

## Отправка удочек

- [ ] Выбор "🎣 Удочки" → список удочек с ценами
- [ ] Пагинация работает (если удочек много)
- [ ] Выбор удочки → запрос username/ID
- [ ] Оплата → удочка выдается получателю через grant_rod с stack=True
- [ ] Проверка: нет удочки у отправителя → ошибка
- [ ] Проверка: недостаточно монет для покрытия стоимости → ошибка
- [ ] Проверка: временная удочка стакается правильно

## Отправка наживок

- [ ] Выбор "🪱 Наживки" → выбор локации
- [ ] Выбор локации → список наживок для этой локации
- [ ] Пагинация работает (если наживок много)
- [ ] Выбор наживки → запрос количества
- [ ] Ввод количества → запрос username/ID
- [ ] Оплата → наживка списывается у отправителя, добавляется получателю
- [ ] Проверка: недостаточно наживки → ошибка
- [ ] Проверка: недостаточно монет для покрытия стоимости → ошибка

## Отправка сетей

- [ ] Выбор "🕸️ Сети" → запрос количества
- [ ] Ввод числа → запрос username/ID
- [ ] Оплата → сети списываются у отправителя, добавляются получателю
- [ ] Проверка: недостаточно сетей → ошибка

## Отправка одежды

- [ ] Выбор "👕 Одежда" → список одежды с ценами
- [ ] Пагинация работает (если одежды много)
- [ ] Выбор одежды → запрос username/ID
- [ ] Оплата → одежда добавляется получателю
- [ ] Проверка: нет одежды у отправителя → ошибка
- [ ] Проверка: недостаточно бриллиантов для покрытия стоимости → ошибка

## Поиск получателя

- [ ] Ввод @username → находит пользователя
- [ ] Ввод username без @ → находит пользователя
- [ ] Ввод ID → находит пользователя
- [ ] Несуществующий username → ошибка с подсказкой попробовать ID
- [ ] Несуществующий ID → ошибка
- [ ] Попытка отправить самому себе → ошибка

## Оплата и передача

- [ ] Pre-checkout проверяет наличие ресурсов
- [ ] Pre-checkout проверяет актуальность сессии
- [ ] Недостаточно ресурсов на момент оплаты → оплата отклоняется
- [ ] Успешная оплата → подарок передается
- [ ] Отправитель получает подтверждение с деталями
- [ ] Получатель получает уведомление в DM (если возможно)
- [ ] Повторная оплата того же инвойса блокируется

## Навигация

- [ ] Кнопка "🔙 Главное меню" возвращает в начальное меню
- [ ] Кнопка "❌ Отмена" закрывает сессию
- [ ] Все кнопки пагинации работают корректно
- [ ] Кнопка "🔙 Выбор локации" (для наживок) возвращает к списку локаций

## Крайние случаи

- [ ] Истекшая сессия → ошибка с просьбой начать заново
- [ ] Чужая кнопка (другой user_id) → игнорируется или показывается ошибка
- [ ] Получатель удалил профиль между выбором и оплатой → корректная обработка
- [ ] Получатель заблокировал бота → подарок отправляется, но уведомление не приходит
- [ ] Callback_data слишком длинный → используется короткий формат

## Интеграция с существующими системами

- [ ] `/cancel` отменяет и RAF, и /send корректно
- [ ] Система стакания удочек работает как ожидается
- [ ] База данных обновляется правильно (монеты, сети, наживки, удочки, одежда)
- [ ] Логирование ошибок работает
- [ ] Транзакции Telegram Stars записываются корректно

## Производительность

- [ ] Большие списки (удочки, одежда, наживки) загружаются быстро
- [ ] Пагинация не вызывает задержек
- [ ] База данных не блокируется при передаче

## Безопасность

- [ ] Нельзя отправить больше ресурсов, чем есть
- [ ] Нельзя отправить отрицательное количество
- [ ] Payload корректно парсится и валидируется
- [ ] Защита от дублирования платежей работает
